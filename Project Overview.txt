Project Overview:

What is the main problem this project solves?
It solves the problem of master data management, taking messy data from different sources and creating a single, reliable "Golden Record" for a student 

The Problem (The "Messy" Data)
​Imagine the Placement Cell receives three different files from three different departments. Notice how inconsistent they are:
​Source A (Exam Cell - CSV): Uses roll_no, cgpa, and backlogs.
​Source B (CS Department - JSON): Uses id, name, and lists completed_courses.
​Source C (Student Self-Portal - API/JSON): Uses student_email, and a list of skills.
​None of these systems share the exact same keys, and the data types might be different (e.g., a CGPA might be a string in one and a float in another)

The Solution: ​Your pipeline's job is to read these three sources, clean them, and merge them into a single schema.


What are the objectives/goals?
- to provide a single clean schema from which the school can know which students are eligible for which placement (company) and to what degree, what skills do they possess and what skills they lack, 
- to reduce the time in grading students and assigning them to the placements that they are eligible for
- for companies to easily get a list of all the student that are eligible along with their skills and cgpa

Who are the end users? (e.g., placement office, students, companies)
- placement office can see the data
- students can see the skills they lack and howfar they are from being eligible for a placement
- companies see which students are eligible for which roles

Methodology:
What data preprocessing techniques do you use?
-- data cleaning which is done after ingestion where each source is cleaned separately but the modules used are reusable for different file sources so no need to write redundant modules for each source file only specific cleaning files. Removing punctuations, lower casing all data, normalize_text_columns, convert_[numeric,boolean]_columns, drop_duplicate_rows, handle_missing_values by filling not dropping rows, validate_column_values if they are valid
-- data transformation: converting strings to lists on skills etc, new columns such as full_name from first name and last name, pd_to_numeric to ensure integers are really integers, using same department name i.e CSE, CS, Btech cse --> CSE
-- integration: creating a unified schema view of the data by merging all the df of the different sources on student_id except the company_rules_df

How do you handle missing values and outliers?
-- outliers we clip them, i.e cgp is 0 to 10, anything below or above is clipped to 0 or 10 respectivels, attendance, readiness score is clipped between 0 and 100, current_semester is clipped if above 8

Any specific validation/testing approaches?
-- testing approaches i used are bottom up, unit, integration/system testing, white box testing
